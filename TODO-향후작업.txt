 콘텐츄어 - 향후 작업 목록
 작성일: 2026-03-23

[1] PortOne 실결제 전환 (우선순위: 높음)
--------------------------------------
- 현재: 테스트 모드 (토스페이먼츠 테스트 채널)
- 필요: 사업자등록 후 PortOne 실결제 채널 생성
- 작업:
  1) PortOne 관리자(admin.portone.io) > 실결제 채널 생성
  2) payment-data.js의 Store ID / Channel Key를 실결제 값으로 교체
  3) Supabase 환경변수에 PORTONE_API_SECRET 설정
     (Supabase 대시보드 > Settings > Edge Functions > Secrets)
  4) verify-payment Edge Function이 실결제 검증 자동 처리

[2] 이메일 알림 구현 (우선순위: 높음)
--------------------------------------
- 현재: DB 알림만 저장, 외부 발송 없음
- 추천 서비스: Resend (resend.com) - 무료 100건/일
- 작업:
  1) Resend 가입 후 API Key 발급
  2) Supabase 환경변수에 RESEND_API_KEY 설정
  3) send-notification Edge Function에 이메일 발송 로직 추가
  4) 알림 트리거 시점:
     - 새 문의 접수 시 관리자에게
     - 계약 상태 변경 시 고객사/통역사에게
     - 결제 완료 시 관리자에게
     - 배정 요청 시 통역사에게

[3] 카카오 알림톡 연동 (우선순위: 중간)
--------------------------------------
- 현재: 보류 (사업자 인증 필요)
- 작업:
  1) Solapi 또는 Aligo에서 사업자 인증 완료
  2) API Key 발급
  3) 카카오톡 채널 생성 + 연동
  4) 메시지 템플릿 등록 (카카오 승인 1~3일 소요)
  5) send-notification Edge Function에 알림톡 발송 로직 추가

[8] 노쇼 알림 시스템 (우선순위: 높음)
--------------------------------------
배경: 통역사 당일 노쇼는 회복 불가능한 운영 리스크 (특히 해외 전시).
2026-05-06에 Phase 1 완료, 테스트 검증까지 끝남. Phase 2~4는 보류 중.

[완료] Phase 1 — D-day 컨펌 수동 노출 (2026-05-06)
  - DB: 42_통역계약.confirmation_clicked_at 컬럼 (마이그레이션 적용 완료)
  - 통역사 대시보드 홈: D-2 ~ 당일 미확인 계약 빨간 배너 + "✅ 확인 완료" 버튼
  - 어드민 계약 관리: D-1 ~ 당일 미확인 계약 빨간 노쇼 위험 알림 카드
  - 고객사 계약 카드: D-2 ~ 당일 ✅녹색/⏳노란색 컨펌 상태 뱃지
  - 더미 계약으로 어드민 화면 동작 확인 + 정리 완료

[Phase 2] 자동 알림 시스템 (보류 — 결정 후 재개)
  목표: 사람이 화면을 보지 않아도 자동으로 알림이 가게 함
  - Resend로 D-3/D-2/D-1 자동 이메일 발송 (전시 정보 + 컨펌 안내)
  - D-1 21:00에 미확인 시 어드민에게 자동 이메일/SMS
  - 만들 파일:
    * api/cron-confirmation-reminders.js (통역사용)
    * api/cron-noshow-alert.js (어드민용)
    * vercel.json에 cron schedule 등록
    * Vercel 환경변수 CRON_SECRET 추가

  [재개 전 결정 필요 사항]
  1) Vercel 플랜 (Hobby vs Pro)
     - Hobby: cron 최대 2개, 매일 1회만 가능 (시각 하나)
     - Pro: 시간당·분당도 가능
     - 현재 어떤 플랜인지 확인 필요
  2) Resend 도메인 인증 상태
     - 메모상 noreply@contentour.co.kr 미인증 추정 (28일 전 정보)
     - 미인증이면 외부 이메일 발송 실패 가능성
     - Resend 대시보드 https://resend.com/domains 에서 상태 확인
     - 자체 도메인(contentour.co.kr) 결정과 묶여 있음 — 도메인 결정 후 인증
  3) 발송 시각
     - 단일 cron (09:00 KST) 통합: 리마인더 + 어드민 알림 한 번에
     - 또는 두 cron 분리: 09:00 통역사 리마인더 / 21:00 어드민 알림 (더 효과적)

[Phase 3] 백업 통역사 + 긴급 매칭 (보류)
  목표: 노쇼 발생 시 즉시 대체 인력 투입
  - 모든 계약에 백업 통역사 사전 지정 (1순위 + 2순위, 같은 국가/언어)
  - 백업 통역사 스탠바이 수당 정책 결정 (정상 단가의 20~30% 등)
  - 1순위 미확인 → 자동으로 2순위 활성화 + 카톡 알림
  - 어드민에 긴급 통역사 풀 조회 (국가·언어·가용 자동 필터)
  - 정책 결정: 해외 전시 백업 의무 vs 선택, 비용 반영 방법

[Phase 4] 노쇼 사후 처리 자동화 (보류)
  목표: 약관(통역사 약관 제5조) 자동 집행
  - "노쇼 신고" 버튼 → 통역사 패널티 카운터 자동 증가
    1회: 경고 + 위약금 / 2회: 30일 정지 / 3회: 영구 이용 제한
  - 자동 환불 처리 (고객사) + 통역사 정산 차감
  - 분쟁 조정 페이지 (통역사 이의 제기 + 어드민 검토)
  - 카톡/SMS로 통역사 본인에게 패널티 통보

[보너스] 사전 예방 정책 (코드 외)
  - 업무 SOP 문서화: 어드민의 D-2 카톡 컨펌, D-1 전화 (위험 케이스만)
  - 위험 케이스 정의: 해외 + 희귀 언어 / 첫 거래 통역사 / 고액 계약 / 무응답
  - 백업 통역사 사전 지정 워크플로 정립
