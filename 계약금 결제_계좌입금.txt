  콘텐츄어 - 가상계좌 결제 연동 설정 가이드
  작성일: 2026-03-30
══════════════════════════════════════════════════════════════

■ 현재 상태
  - PortOne V2 SDK 로드 완료 (customer-dashboard.html)
  - payment-data.js에 결제 로직 구현 완료 (카드/계좌이체/가상계좌)
  - Store ID, Channel Key 설정됨 (테스트/실거래 확인 필요)


══════════════════════════════════════════════════════════════
  STEP 1. PortOne 관리자 콘솔 가입 및 로그인
══════════════════════════════════════════════════════════════

  1) https://admin.portone.io 접속
  2) 회원가입 또는 로그인
  3) "상점" 메뉴에서 현재 Store ID 확인
     → 현재 코드에 설정된 값: store-f2f2a330-69de-4774-bd36-4ed70b5080ca
     → 이 값이 본인 상점과 일치하는지 확인


══════════════════════════════════════════════════════════════
  STEP 2. PG사 계약 (가상계좌 지원 PG사 선택)
══════════════════════════════════════════════════════════════

  가상계좌를 지원하는 주요 PG사:
  ┌─────────────────┬──────────────┬────────────────────┐
  │ PG사            │ 수수료(약)   │ 특징               │
  ├─────────────────┼──────────────┼────────────────────┤
  │ 토스페이먼츠    │ 건당 300원   │ 가입 간편, 추천    │
  │ KG이니시스      │ 건당 300원   │ 대형 PG, 안정적    │
  │ NHN KCP         │ 건당 300원   │ 가상계좌 특화      │
  │ 나이스페이먼츠  │ 건당 300원   │ 다양한 은행 지원   │
  └─────────────────┴──────────────┴────────────────────┘

  계약 시 필요 서류:
  - 사업자등록증 사본
  - 대표자 신분증 사본
  - 통장 사본 (정산 입금용)
  - 서비스 URL (https://contentour-landing.vercel.app)

  ※ PG사 심사 소요: 보통 3~7 영업일


══════════════════════════════════════════════════════════════
  STEP 3. PortOne에서 결제 채널 생성
══════════════════════════════════════════════════════════════

  1) PortOne 관리자 콘솔 → "결제 연동" → "결제 채널 관리"
  2) "채널 추가" 클릭
  3) STEP 2에서 계약한 PG사 선택
  4) PG사에서 발급받은 키 입력:
     - MID (가맹점 ID)
     - API Key / Secret Key
  5) 결제 수단에서 "가상계좌" 체크 활성화
  6) 채널 저장 → Channel Key 복사

  ※ 테스트 채널과 실거래 채널을 각각 만들 수 있음
     - 먼저 "테스트" 채널로 동작 확인 후
     - "실거래" 채널로 전환 권장


══════════════════════════════════════════════════════════════
  STEP 4. 코드에 키 값 반영
══════════════════════════════════════════════════════════════

  파일: payment-data.js (라인 8~9)

  현재:
    PORTONE_STORE_ID: 'store-f2f2a330-69de-4774-bd36-4ed70b5080ca',
    PORTONE_CHANNEL_KEY: 'channel-key-0141bb27-c39f-4ea5-a78d-144d8e6dd6f5',

  변경:
    PORTONE_STORE_ID: 'STEP 1에서 확인한 Store ID',
    PORTONE_CHANNEL_KEY: 'STEP 3에서 생성한 Channel Key',

  ※ 테스트 시에는 테스트 Channel Key, 실거래 시에는 실거래 Channel Key 사용


══════════════════════════════════════════════════════════════
  STEP 5. 가상계좌 입금 확인 웹훅(Webhook) 설정
══════════════════════════════════════════════════════════════

  가상계좌는 고객이 나중에 입금하므로, 입금 완료 시
  자동으로 알림을 받을 수 있도록 웹훅을 설정해야 함.

  1) PortOne 관리자 → "결제 연동" → "웹훅 관리"
  2) 웹훅 URL 등록:
     → https://yvtgfieoeoqhfbofzunk.supabase.co/functions/v1/portone-webhook
     (Supabase Edge Function으로 처리)
  3) 이벤트 선택:
     - payment.paid (결제 완료 = 입금 확인)
     - payment.cancelled (결제 취소)
     - payment.failed (결제 실패)

  ※ 웹훅이 설정되어야 가상계좌 입금 시 자동으로
     DB 상태가 "입금 대기" → "결제 완료"로 변경됨


══════════════════════════════════════════════════════════════
  STEP 6. Supabase Edge Function 배포 (웹훅 수신용)
══════════════════════════════════════════════════════════════

  PortOne에서 입금 확인 웹훅을 보내면 이를 수신하여
  DB를 업데이트하는 서버 함수가 필요함.

  필요한 Edge Function:
  - portone-webhook: 입금 확인 → 계약 상태 업데이트
  - verify-payment: 결제 검증 (이미 코드에 참조됨)

  ※ 이 부분은 별도 개발 작업이 필요하며,
     준비되면 Claude에게 요청하면 구현 가능


══════════════════════════════════════════════════════════════
  STEP 7. 테스트 진행
══════════════════════════════════════════════════════════════

  1) 테스트 채널 키로 payment-data.js 설정
  2) 고객사 대시보드에서 계약금 결제 → 가상계좌 선택
  3) PortOne 테스트 결제창에서 가상계좌 발급 확인
  4) 테스트 입금 처리 (PortOne 관리자에서 수동 처리 가능)
  5) 웹훅으로 입금 확인 → DB 상태 변경 확인
  6) 정상 동작 확인 후 → 실거래 채널 키로 교체


══════════════════════════════════════════════════════════════
  STEP 8. 실거래 전환
══════════════════════════════════════════════════════════════

  1) payment-data.js의 Channel Key를 실거래 키로 교체
  2) 배포 (vercel --prod)
  3) 실제 소액 결제 테스트 (1,000원 등)
  4) 정상 확인 후 서비스 운영 시작
