■ 5. 위약금 제도 전체 구현 (5단계)
  [1단계] DB 테이블 생성
    - 50_위약금정책: 취소 시점별 위약금 비율 (고객/통역사 각 5건)
    - 51_취소내역: 취소 신청/처리 이력
    - 42_통역계약: cancelled_by, cancel_reason, cancelled_at 컬럼 추가
    - 40_통역사프로필: penalty_count, is_suspended, suspended_until 컬럼 추가
    - calculate_penalty() DB 함수: 위약금 자동 계산
    - RLS 정책 설정

  [2단계] 고객 대시보드 — 취소 신청 UI
    - 계약 상세 모달에 '계약 취소 신청' 버튼
    - 취소 모달: 위약금 자동 계산(DB 함수), 사유 선택
    - 51_취소내역 기록 + 42_통역계약 cancelled 처리

  [3단계] 통역사 대시보드 — 파견 취소
    - 계약 상세에 '파견 취소 신청' 버튼
    - 패널티 정책 안내 (경고/제한/정지)
    - 패널티 누적 시 penalty_count 증가, 3회 이상 매칭 정지

  [4단계] 관리자 대시보드 — 취소 관리
    - 사이드바 '⚠️ 취소 관리' 메뉴 추가
    - 취소 관리 뷰: 요약 카드, 필터(전체/대기/승인/환불/거절)
    - 승인/거절 기능 (거절 시 계약 상태 복구)

  [5단계] FAQ 위약금 정책 공지
    - FAQ '일정 변경이나 취소가 가능한가요?' 항목 업데이트
    - 취소 시점별 위약금 테이블 추가

■ 6. 성과사례 페이지 개편
  - 대표 고객사례를 통역 중심으로 재구성
    · 태그: 부스시공 → 부스 통역/현장 통역/미팅 통역
    · 지표: 통역사 이름 + 상담 통역 건수 + 산업 분야

  - DB 연동 (52_성과사례 테이블)
    · 관리자 대시보드에서 CRUD (등록/수정/삭제/공개 관리)
    · 성과사례 페이지에서 DB 동적 로드

  - 관리자 폼 디자인 개편
    · 이미지: URL 입력 → 클릭 업로드 (Supabase Storage case-images 버킷)
    · 섹션별 색상 구분 (기본정보/통역사정보/성과지표/추가설정)
    · 이미지 미리보기 + 제거 기능

  - 상세 페이지 (case-detail.html) 신규 생성
    · 대표 이미지, 성과 지표 카드, 업무 내용 요약
    · 통역사 프로필 + 현장 코멘트
    · 현장 사진 슬라이드 (좌우 화살표 + 도트 네비 + 키보드)
    · 프로젝트 담당자 정보
    · 풀스크린 이미지 뷰어

  - 관리자 폼 상세 정보 추가
    · 통역사 코멘트, 업무 요약, 담당자명/직책
    · 현장 사진 여러 장 업로드

■ 7. 통역사 대시보드 프로필 사진 업로드 수정
  - InterpreterApp.currentUser가 null일 때 폴백 추가
  - ContentourAuth.getCurrentUser()로 직접 세션 확인
  - 프로필 사진, 증빙서류, 서류 로드 모두 동일 패턴 적용

■ 8. 국가별 통역사 네트워크 DB 연동
  - 하드코딩 데모 통역사 전부 삭제
  - DB(40_통역사프로필)에서 is_active=true 통역사만 동적 로드
  - 40_통역사프로필에 country_code, field_tag, cases_count, rating, satisfaction 컬럼 추가
  - 통역사 없는 국가: '모집 중' 안내 + 지원 링크 표시
  - 전체 통역사 보기(interpreters.html)도 동일 적용
