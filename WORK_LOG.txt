▶ 7. 프로젝트 보안 점검 (3개 팀 병렬 분석)
──────────────────────────────────────────────
프론트엔드/UX팀, 백엔드/보안팀, DB/아키텍처팀 3개 팀으로
프로젝트 전체 점검 수행. 총 23개 문제 발견.

[1주차 긴급 수정 — 완료]
• Supabase RLS 점검: 전체 테이블 RLS 활성화 확인
  → 48_통역사지원서의 위험한 anon ALL 정책 제거 (마이그레이션 적용)
• 데모 모드 보안 강화: demoToken 서명 검증 추가
  → sessionStorage 직접 조작으로 대시보드 접근 불가하도록 차단
• XSS innerHTML 방지: escHtml() 유틸 함수 추가
  → interpreter-app.js 내 DB 데이터 렌더링 17곳에 이스케이프 적용

[2~3주차 단기 수정 — 완료]
• 결제 입력값 검증: verifyAndSavePayment에 contractId, amount,
  paymentType, merchantUid 유효성 검사 추가
• 채팅 접근 제어: loadMessages에 계약 참여자 확인 로직 추가
  (서버 RLS + 클라이언트 이중 보호)
• 반응형 브레이크포인트 통일: 1024px / 768px / 480px
  → customer-dashboard, admin-dashboard, interpreter-dashboard
  → 캘린더 최소 폰트 0.5rem → 0.65rem (가독성 개선)
  → CSS 변수 중복 제거 (--gray-* → --g-* 통합)
• 다중 테이블 에러 핸들링: interpreter-data.js saveProfile에서
  두 번째 UPDATE 에러도 체크하도록 수정

[4주차 중기 개선 — 완료]
• 데이터 접근 계층 통합: shared-constants.js 생성
  → CT 네임스페이스로 상태 매핑, 유틸 함수, 감사 로그 통합
  → 3개 대시보드에 script 태그 추가

  → 비밀번호 찾기 죽은 링크 수정
