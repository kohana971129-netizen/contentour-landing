# 맥북에서 contentour-landing 작업하기 — 셋업 가이드

> 출장용. 회사 PC와 동일하게 코드 수정 → push → Vercel 자동배포까지 가능.
> GitHub: https://github.com/kohana971129-netizen/contentour-landing
> 배포 사이트: https://contentour-landing.vercel.app

---

## ✅ 핵심 요약 (먼저 읽기)
- 챙겨갈 **비밀 파일 없음**. 진짜 키들은 전부 Vercel 대시보드에 있음.
- 맥북에서 할 일: **Git + Node.js + Claude Code 설치 → 코드 클론 → npm install**.
- 평소 작업(코드 고치고 → push → 배포 사이트에서 확인)은 맥북에서 그대로 됨.

## 1단계. 필수 도구 설치

### (1) Homebrew (맥 패키지 관리자) — 없으면 설치
터미널 앱(Spotlight에서 "터미널" 검색) 열고:
```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```
설치 끝나면 화면 안내대로 PATH 추가 명령 한두 줄을 복사해 실행 (Apple Silicon이면 보통):
```bash
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
eval "$(/opt/homebrew/bin/brew shellenv)"
```

### (2) Git
```bash
brew install git
git --version   # 버전 나오면 성공
```

### (3) Node.js (LTS)
```bash
brew install node
node -v    # v20 이상 권장
npm -v
```

### (4) Claude Code
```bash
npm install -g @anthropic-ai/claude-code
claude --version
```
> 처음 실행 시 `claude` 입력하면 로그인(Anthropic 계정) 진행됨.

## 2단계. 코드 받기 (클론)

작업 폴더로 이동 후 클론:
```bash
cd ~/Documents        # 원하는 위치
git clone https://github.com/kohana971129-netizen/contentour-landing.git
cd contentour-landing
npm install           # package.json 의존성 설치 (supabase-js, resend 등)
```

> 클론 시 GitHub 로그인 요구하면: 사용자명 + **비밀번호 대신 Personal Access Token(PAT)** 입력.
> PAT 발급: GitHub → Settings → Developer settings → Personal access tokens → repo 권한 체크.
> (또는 `brew install gh && gh auth login` 으로 더 쉽게 인증 가능)

## 🔁 매일 작업 루틴 (두 컴퓨터 충돌 방지 — 제일 중요!)

회사 PC ↔ 맥북을 오가니까 이 순서만 지키면 절대 안 꼬임:

**작업 시작 전 (항상):**
```bash
git pull          # 다른 컴퓨터에서 한 작업 받아오기
```

**작업 끝나면 (항상):**
```bash
git add -A
git commit -m "작업 내용 설명"
git push          # GitHub 올리기 → Vercel 자동배포
```

> Claude Code에게 "push 해줘"라고 말하면 알아서 처리함.
> ⚠️ 한쪽에서 push 안 하고 다른 컴퓨터에서 작업하면 충돌(conflict) 남.
>    **떠나기 전 반드시 push, 시작할 때 반드시 pull.**

## 📦 비밀키(.env)에 대해

- 화면(HTML/CSS/JS) 수정 + push + 배포 사이트 확인 → **키 필요 없음. 바로 됨.**
- 맥북에서 API 함수를 *로컬에서 직접 돌려보려면*(`vercel dev`로 결제·DB 테스트)
  아래 4개 값이 필요한데, 로컬엔 없고 **Vercel 대시보드에 있음**:

  | 환경변수 | 용도 |
  |---|---|
  | `SUPABASE_SERVICE_ROLE_KEY` | Supabase 관리자 권한 |
  | `PORTONE_V2_API_SECRET` | 결제 |
  | `PORTONE_WEBHOOK_SECRET` | 결제 웹훅 |
  | `RESEND_API_KEY` | 이메일 |

  필요해지면: Vercel → 프로젝트 → Settings → Environment Variables 에서 값 복사 →
  프로젝트 루트에 `.env` 파일 만들어 붙여넣기 (이 파일은 git에 안 올라감 = 안전).
  ```
  SUPABASE_SERVICE_ROLE_KEY=...
  PORTONE_V2_API_SECRET=...
  PORTONE_WEBHOOK_SECRET=...
  RESEND_API_KEY=...
  ```

> 프론트엔드용 Supabase URL/anon키는 supabase-config.js에 이미 들어있음(공개 OK).

## 🆘 자주 막히는 곳

| 증상 | 해결 |
|---|---|
| `git push` 시 인증 실패 | PAT 재발급 또는 `gh auth login` |
| `command not found: node/git` | brew 설치 확인, 터미널 새로 열기 |
| `npm install` 에러 | `node -v`로 버전 확인(20+), 재시도 |
| 클론은 됐는데 화면이 안 뜸 | 정적 사이트라 `index.html` 더블클릭하거나 배포 사이트에서 확인 |
| push 했는데 배포 안 됨 | Vercel 대시보드에서 배포 로그 확인 |
