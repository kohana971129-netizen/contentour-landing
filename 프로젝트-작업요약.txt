========================================
 콘텐츄어 통역 운영 시스템 — 전체 작업 요약
========================================

■ 이 프로젝트가 뭔가요?

콘텐츄어는 전시회 통역사를 고객사에 연결해주는 플랫폼입니다.
3가지 대시보드가 있습니다:
  - 관리자 대시보드 — 문의 접수, 통역사 배정, 계약/정산 관리
  - 고객사 대시보드 — 통역 문의, 일정 확인, 계약/결제
  - 통역사 대시보드 — 배정 수락/거절, 일정, 정산 확인


========================================
 1단계: 기반 구축 (초기~3월 중순)
========================================

[코드 정리]
- 인라인 스타일 8,000줄을 CSS 파일로 분리하여 유지보수 쉽게 개선

[디자인 통일]
- 디자인 토큰(색상, 폰트, 간격) 시스템을 만들어 전체 페이지에 적용

[보안 강화]
- XSS 취약점 수정, 데모 비밀번호 노출 제거, 에러 처리 보강

[인증 시스템]
- Supabase 기반 로그인/회원가입/비밀번호 찾기 구현
- 관리자/고객사/통역사 역할 분리

[DB 연동]
- 고객사 프로필 수정, 통역사 프로필 등이 실제 데이터베이스에 저장되도록 연결


========================================
 2단계: 핵심 기능 구현 (3월 23일)
========================================

[결제 시스템 연동]
- PortOne(토스페이먼츠)을 연결하여 고객사가 계약금을 결제할 수 있도록 구현
- 현재 테스트 모드 (실결제 X)

[결제 보안 처리]
- 결제 검증을 프론트엔드가 아닌 서버(Supabase Edge Function)에서 처리
- 결제 위변조 방지

[Google 캘린더 연동]
- 통역사/고객사가 파견 일정을 Google 캘린더에 한 번에 추가할 수 있는 버튼 구현

[마이페이지 DB 연동]
- 고객사가 프로필(이름, 직급, 연락처, 회사명)을 수정하면 실제 DB에 저장

[백엔드 분리]
- 민감한 작업(결제 검증, 관리자 승인, 알림 발송)을 서버 측 함수 3개로 분리
  1) verify-payment — 결제 검증
  2) admin-actions — 관리자 전용 작업 (정산 승인/거절, 계약 상태 변경)
  3) send-notification — 알림 발송

[더미 데이터 정리]
- 테스트용 가짜 계정 8개와 관련 데이터를 DB에서 전부 삭제

[CSS 배포 문제 해결]
- Vercel 배포 시 CSS 파일이 깨지는 문제를 인라인 처리 + CLI 직접 배포로 해결


========================================
 3단계: 데이터 정리 및 품질 개선 (3월 25일)
========================================

[배포 반영 확인]
- 이전 작업이 실제 사이트에 반영 안 된 것을 발견
- Vercel 재배포 + 캐시 무효화 헤더 적용으로 해결

[사이드바 로고 통일]
- 3개 대시보드의 사이드바 상단 로고를 동일한 디자인으로 통일
  : 관리자 — CT + CONTENTOUR + Admin
  : 통역사 — CT + CONTENTOUR + Interpreter
  : 고객사 — CT + CONTENTOUR + Customer Portal
- 클릭 시 홈페이지가 새 탭으로 열림

[더미 통역사 완전 정리]
- DB에서 가짜 통역사 4명 삭제
- 로컬 파일 5개에서 더미 이름 85건을 실제 계정(김하나)으로 교체
- 통역사 관리 페이지: 더미 9명 → 김하나 실계정 1명만 유지

[배정 → 통역사 연동 수정]
- 문제: 관리자가 통역사를 배정해도 통역사 대시보드에서 안 보임
- 원인: 관리자가 배정 시 견적문의 테이블만 업데이트하고, 통역사가 조회하는 계약 테이블에는 데이터가 생성되지 않았음
- 해결: 배정 시 계약 레코드가 자동으로 생성되도록 로직 추가

[계정 관리]
- 로그인 오류 원인 파악 (비밀번호 해시 방식 불일치) 및 재설정


========================================
 현재 시스템 구성
========================================

[홈페이지] contentour-landing.vercel.app
  ├── 고객사 로그인/가입 (client-auth.html)
  ├── 관리자 로그인 (admin-dashboard.html 내장)
  ├── 통역사 지원서 (interpreter-apply.html)
  │
  ├── [고객사 대시보드]

  │

  │     문의 관리 → 통역사 배정 → 견적 발송 → 계약/정산 승인 → 채팅 모니터링
  │
  └── [통역사 대시보드]
        배정 수락/거절 → 일정 확인 → 상담일지 작성 → 정산 요청


========================================
 기술 스택
========================================

- 프론트엔드: HTML / CSS / JavaScript (프레임워크 없이 순수 구현)
- 데이터베이스 + 인증: Supabase (PostgreSQL + Auth + Edge Functions)
- 배포: Vercel (CLI 토큰 방식)
- 결제: PortOne V2 (토스페이먼츠 테스트 채널)
- 외부 연동: Google Calendar API
